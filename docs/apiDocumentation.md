## 1. Shorten a URL

Accepts a long URL and returns a unique short code. If the URL has already been shortened, the existing short code is returned instead of creating a duplicate.

> **Note:** This endpoint is rate-limited to **5 requests per minute** per IP address. Exceeding this limit returns a `429` response with a `Retry-After` header.

**Endpoint**
```
POST /api/shorten
```

**Request Headers**
| Header | Value |
|---|---|
| `Content-Type` | `application/json` |

**Request Body**
| Field | Type | Required | Description |
|---|---|---|---|
| `originalUrl` | `string` | ✅ | The full URL to shorten. Must begin with `http://` or `https://`. |

**Example Request**
```json
{
  "originalUrl": "https://www.example.com/some/very/long/path"
}
```

**Success Response — New URL Created**
- **Status:** `201 Created`
```json
{
  "msg": "URL shortened successfully",
  "shortUrl": "a1b2c3"
}
```

**Success Response — URL Already Exists**
- **Status:** `200 OK`
```json
{
  "msg": "URL already shortened",
  "shortUrl": "a1b2c3"
}
```

**Error Responses**
| Status | Reason |
|---|---|
| `400 Bad Request` | `originalUrl` is missing, not a string, or not a valid URL. |
| `429 Too Many Requests` | Rate limit exceeded. Includes a `Retry-After` header (seconds). |

## 2. Redirect via Short Code

Resolves a short code to its original URL and redirects the client. Also records a click event for analytics purposes.

**Endpoint**
```
GET /:shortCode
```

**URL Parameters**
| Parameter | Type | Required | Description |
|---|---|---|---|
| `shortCode` | `string` | ✅ | The short code generated by the shorten endpoint. |

**Example Request**
```
GET /a1b2c3
```

**Success Response**
- **Status:** `302 Found`
- The client is redirected to the original URL. No JSON body is returned.

**Error Responses**
| Status | Reason |
|---|---|
| `404 Not Found` | No URL exists for the provided short code. |

## 3. Get Recent Links

Returns the 10 most recently shortened URLs, ordered by creation date (newest first). Each entry includes the total click count.

**Endpoint**
```
GET /api/shorten/recent
```

**Request Body**
None.

**Success Response**
- **Status:** `200 OK`

```json
{
  "msg": "Recent Links",
  "links": [
    {
      "shortCode": "a1b2c3",
      "originalUrl": "https://www.example.com/some/very/long/path",
      "status": "ACTIVE",
      "clicks": 14
    },
    {
      "shortCode": "f4e5d6",
      "originalUrl": "https://www.google.com",
      "status": "ACTIVE",
      "clicks": 3
    }
  ]
}
```

**Response Fields**
| Field | Type | Description |
|---|---|---|
| `shortCode` | `string` | The unique short code for this URL. |
| `originalUrl` | `string` | The original long URL. |
| `status` | `string` | Always `"ACTIVE"` in the current implementation. |
| `clicks` | `number` | Total number of times this short URL has been visited. |

## 4. Get URL Analytics

Returns detailed analytics for a specific short URL, including total click count and a day-by-day click history for the past 7 days.

**Endpoint**
```
GET /api/analytics/:shortCode
```

**URL Parameters**
| Parameter | Type | Required | Description |
|---|---|---|---|
| `shortCode` | `string` | ✅ | The short code to retrieve analytics for. |

**Example Request**
```
GET /api/analytics/a1b2c3
```

**Success Response**
- **Status:** `200 OK`

```json
{
  "msg": "Short URL Analytics",
  "shortCode": "a1b2c3",
  "originalUrl": "https://www.example.com/some/very/long/path",
  "createdAt": "2026-04-20T08:30:00.000Z",
  "totalClicks": 17,
  "clicksSeries": [2, 0, 5, 3, 1, 4, 2]
}
```

**Response Fields**
| Field | Type | Description |
|---|---|---|
| `shortCode` | `string` | The short code that was looked up. |
| `originalUrl` | `string` | The original long URL. |
| `createdAt` | `string` | ISO 8601 timestamp of when the short URL was created. |
| `totalClicks` | `number` | Total number of clicks recorded across all time. |
| `clicksSeries` | `number[]` | An array of 7 integers representing click counts per day for the last 7 days (oldest to newest). |

**Error Responses**
| Status | Reason |
|---|---|
| `404 Not Found` | No URL exists for the provided short code. |

## Error Responses

All errors follow a consistent structure:

```json
{
  "msg": "A human-readable error message"
}
```

**Standard Error Status Codes**

| Status Code | Meaning |
|---|---|
| `400 Bad Request` | The request was malformed or missing required fields. |
| `404 Not Found` | The requested resource does not exist. |
| `429 Too Many Requests` | The client has exceeded the rate limit. Check the `Retry-After` header for the number of seconds to wait before retrying. |
| `500 Internal Server Error` | An unexpected server error occurred. |
